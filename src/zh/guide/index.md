# 指南

本目录用于放“如何使用/如何维护这本书”的内容，目标是降低协作成本，让新加入的人能快速进入状态。

## 这个目录放什么

- **上手与预览**：如何安装 mdBook、如何本地预览、如何构建
- **写作规范**：标题层级、代码块、链接、图片、命名约定
- **协作约定**：评审流程、如何提交文档、如何拆分章节

## 这个目录不放什么

- 具体功能的设计方案（放到 `design/`）
- 最终决策的结论（放到 `adr/`）
- 会议纪要、过程性讨论（放到 `discussions/`）

## 命名建议

- 文件名：英文小写 + `-`（例如 `writing-style.md`）
- 标题：用中文更易读，必要时加英文术语
